### 计算机基础

1. 所有进程管理的思想都是使得拥有不同资源的不同进程同步，只要资源可以被多个进程同时使用，那么它就可能会发生两个问题`死锁`和`饿死`。
2. 死锁就是进程 A 占用了 file1，还要用 file2，进程 B 占用了 file2，还要用 file1，会形成死锁。
3. redis 的内存回收，类似于 LRU 算法，把高频的留下，把低频的往后。最后用的多了的情况下，低频的就滚蛋了
4. java 中类似于 LRU 这样的链表数据结构叫 linkedhashmap， 继承自 hashMap，多了 before,after 等，形成双向循环链表
5. hashmap 的实现机制

    在 Java1.8 以后，HashMap 在数组、链表的基础上又增加了红黑树的数据结构。在一个数组位置的链表长度大于 8 时数据结构转换为红黑树的结构。

6. 当两个对象的 HashCode 相同会发生什么？

    因为 hashcode 相同，所以它们的 bucket 位置相同，‘碰撞’会发生。因为 HashMap 使用链表存储对象，这个 Entry(包含有键值对的 Map.Entry 对象)会存储在链表中。

7. 为什么 jdk1.8 以前用的是头插法，后面改成尾插法？

    因为以前是数组+链表的结构，没有引入红黑树，通常用户新插的数据使用率很高，所以用头插法效率会很高。但是头插法会造成死循环，两个线程一起操作的话

    而后面引入红黑树后，尾插法效率就不会那么低了，虽然不会死循环，但是会丢数据。

    因为他们都不是线程安全的，如果要安全要用 concurrentHashMap,会在操作的时候，锁那一小部分区域。从而不会出问题。

8. 如何通过 ssh 的形式拉仓库

    通过 ssh key 的形式，相当于我把我这台电脑的身份发给了 github，后续交互的时候就不需要输入密码了。

    1. 检查 sshkey 是否存在 `ls -al ~/.ssh` ,存在的话看里面是否有 `id_rsa.pub`
    2. 如果没有的话生成新的 `ssh-keygen -t rsa -C "your_email@example.com"`
    3. 复制 rsa 到 git 服务器上 `pbcopy < ~/.ssh/id_rsa.pub`

11. **进程 vs 线程**

    - **进程**是操作系统分配资源的最小单位，拥有独立的内存空间。进程间相互隔离，一个进程崩溃不影响其他进程。
    - **线程**是 CPU 调度的最小单位，同一进程内的多个线程共享内存空间和文件句柄等资源。线程创建开销比进程小得多。
    - **通信方式**：进程间通信（IPC）需要借助管道、消息队列、共享内存、Socket 等；线程间可以直接读写共享内存，但需要加锁保护。

    | 对比 | 进程 | 线程 |
    |------|------|------|
    | 内存 | 独立 | 共享 |
    | 创建开销 | 大 | 小 |
    | 崩溃影响 | 不影响其他进程 | 可能拖垮整个进程 |
    | 通信 | IPC（复杂）| 共享内存（需加锁）|

    **前端关联**：Chrome 每个 Tab 是一个独立进程（多进程架构），一个 Tab 崩溃不影响其他 Tab。Node.js 是单进程单线程，但可以用 `child_process` 或 `cluster` 模块开启多进程充分利用多核 CPU。

12. **并发 vs 并行**

    - **并发（Concurrency）**：同一时间段内处理多个任务，但某一时刻只有一个任务在执行。通过快速切换任务产生"同时运行"的假象。单核 CPU 处理多任务就是并发。
    - **并行（Parallelism）**：同一时刻真正有多个任务同时执行，需要多核 CPU 或多处理器。

    ```
    并发：一个厨师做多道菜，炒完A菜切B菜，快速切换
    并行：多个厨师同时各做一道菜
    ```

    **Node.js 的并发模型**：单线程 + 事件循环实现并发。JS 代码本身是单线程跑的，但 I/O 操作（网络、文件）交给底层的 libuv 线程池异步处理，回调结果通过事件循环回到主线程。所以 Node.js 是并发的但不是并行的（单 Worker 的情况下）。

    **AI 训练的并行**：大模型训练需要数千张 GPU 同时运算，是真正的并行 —— 数据并行（不同数据喂给不同 GPU）和模型并行（模型太大，拆开放到不同 GPU）。

13. **I/O 模型**

    I/O 操作（读文件、网络请求）分为两个阶段：**等待数据就绪** 和 **将数据从内核复制到用户空间**。

    - **阻塞 I/O**：调用后线程挂起等待，直到 I/O 完成才返回。简单但浪费 CPU。
    - **非阻塞 I/O**：调用立即返回，数据没好就返回 EAGAIN，需要不断轮询（polling）。CPU 利用率高但轮询本身消耗资源。
    - **I/O 多路复用**（select/poll/epoll）：用一个线程监听多个 I/O 事件，哪个好了处理哪个。Node.js 底层 libuv 在 Linux 上就是用 **epoll** 实现的，这是 Node.js 能高并发的核心原因。
    - **异步 I/O**：内核完成 I/O 后主动通知用户，全程不阻塞。

    **epoll vs select/poll**：
    - `select/poll`：每次调用都需要把所有 fd 从用户态复制到内核态遍历，O(n)
    - `epoll`：只在注册时复制一次，内核用红黑树管理，事件触发时只返回就绪的 fd，O(1)，适合大量连接

    **Node.js 高并发的本质**：不是因为 JS 跑得快，而是 I/O 操作非阻塞，主线程不用等待，可以一直处理新的请求。CPU 密集型任务（加密、图像处理）反而是 Node.js 的弱项。

14. **内存分区：栈 vs 堆**

    程序运行时内存主要分为：

    | 区域 | 存什么 | 特点 |
    |------|--------|------|
    | **栈（Stack）** | 函数调用帧、局部变量、基础类型值 | 自动分配释放，LIFO，空间小（几MB）|
    | **堆（Heap）** | 动态分配的对象、引用类型 | 手动或 GC 管理，空间大，但分配慢 |
    | **代码段** | 可执行代码 | 只读 |
    | **数据段** | 全局变量、静态变量 | 程序生命周期内存在 |

    **JS 中的体现**：
    - 基础类型（number、string、boolean、null、undefined）存在**栈**里，按值传递
    - 对象、数组、函数存在**堆**里，变量持有的是堆地址（引用传递）
    - 函数调用产生调用栈帧，递归过深导致 "Maximum call stack size exceeded" 就是栈溢出

    **为什么对象要存堆里**：大小不固定，运行时才知道，只能动态分配在堆上。栈的分配必须在编译期确定大小。

### AI 计算机基础

1. **为什么 AI 训练用 GPU 而不是 CPU？**

    核心原因：**矩阵乘法的大规模并行**。

    | 对比 | CPU | GPU |
    |------|-----|-----|
    | 核心数 | 几个 ~ 几十个强核 | 数千个弱核（CUDA Core）|
    | 擅长 | 复杂逻辑、串行任务 | 简单运算的大规模并行 |
    | 缓存 | 大（L1/L2/L3）| 小（带宽优先）|
    | 典型场景 | 操作系统、数据库 | 图形渲染、AI 训练/推理 |

    神经网络的核心计算是矩阵乘法（线性变换），这天然适合并行 —— 1000 行 × 1000 列的矩阵乘法，每个输出元素都可以独立计算，GPU 的几千个核同时算，效率远超 CPU 串行。

    **CUDA**：Nvidia 推出的 GPU 并行计算平台，PyTorch/TensorFlow 底层都依赖 CUDA。这也是为什么 AI 领域 Nvidia 一家独大。

2. **大模型推理为什么慢？显存瓶颈**

    LLM 推理（生成文字）的主要瓶颈不是计算量，而是**显存带宽**：
    - 每生成一个 token，都要把整个模型权重从显存读一遍（几十 GB）
    - 带宽跑满了，再强的算力也没用
    - 这就是为什么 70B 模型比 7B 模型慢很多，即使算力充足

    **KV Cache**：推理时把已生成 token 的 Key/Value 缓存下来，避免重复计算。是推理优化的标配，但会占用大量显存，这也是为什么对话轮次很长时会变慢。

3. **量化（Quantization）**

    把模型权重从 FP32（32位浮点）压缩到 FP16、INT8、INT4，减小体积和显存占用，加快推理速度，精度损失可控。
    - FP32 → FP16：体积减半，精度略降，主流做法
    - INT8/INT4：体积缩至 1/4 ~ 1/8，精度有损但消费级显卡可跑大模型
    - `llama.cpp` 的 GGUF 格式就是各种量化级别的本地模型格式

### 思想

1. 什么是囚徒困境

    两个犯人，A,B。

    他们坐牢的情况如下，如果都选择抗拒，全只判一年，如果都选择招供，全判 5 年，如果一个选择招供，一个选择抗拒，抗拒的 10 年，招供的 0 年。

    类比于当前的社会内卷上班等等。

    囚犯无法信任彼此，那么 A 的选择如下，如果选择抗拒，就是 1 + 10 = 11 年。 如果选择招供，就是 5 + 0 = 5 年。虽然抗拒的 1 年很好。但是无法信任别人的情况下，风险也很高，所以最重肯定都是招供，就是都是 5 年。

    就像现在的社会一样，如果大家都抵制 996，可能工资都是 20k. 如果有一个人抵制，另外的拥抱，结果抵制的 10K，拥抱的有 30K。如果都拥抱 996，形成内卷，18K 全是。结果就是大家都拥抱，卷起来。
